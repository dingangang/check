Navbar

一个响应式的水平导航条，支持图片、链接、按钮和下拉菜单

{% include meta.html new=true since="0.4.3" colors=true sizes=false variables=true %}

新的navbar已经取代了弃用的nav组件，你可以点击这里查看老版本的导航条

navbar组件是响应式和多用的水平导航条，其结构如下：

  • navbar主容器
    • navbar-brand位于左侧,，一直可见，通常会包含logo或者其他可选的链接与图标
      • navbar-burger汉堡图标 在移动设备上来控制导航条的显示
    • navbar-menu位于右侧，移动设备时隐藏，PC端则会显示
      • navbar-start菜单的左部分，PC端时，会出接着navbar-brand出现。
      • navbar-end菜单的右部分 ，会出现在导航条的尾部。
        • navbar-item导航条的单个项目，可以是a或者是一个div
          • navbar-link一个链接，带有箭头，作为下拉菜单的毗邻修饰
          • navbar-dropdown下拉菜单，可以包括navbar-item和分隔线
            • navbar-divider水平分隔线来分隔navbar-item
{% include anchor.html name="导航牌（Navbar brand）" %}

navbar-brand是位于导航的左侧，它可以包括：

  • 很多的navbar-item
  • navbar-burger作为最后一个子元素

  • navbar-start
  • navbar-end

navbar-item是可以重复的元素，它可以是：

  • 一个导航链接 {% highlight html %}{{ navbar_item_link_example }}{% endhighlight %}
  • 一个品牌Logo的容器 {% highlight html %}{{ navbar_item_brand_example }}{% endhighlight %}

它可以是一个链接标签<a>或者<div>，作为以下元素的直接子元素：

  • navbar
  • navbar-brand
  • navbar-start
  • navbar-end
  • navbar-dropdown

你可以添加is-expanded修饰器来转变其为一个全宽的元素。

通过CSS或JavaScript来显示/隐藏下拉菜单

navbar-dropdown在移动端是直接可见的{% include bp/touch.html %}但是在PC端时会隐藏{% include bp/desktop.html %}。如何 控制其展示与否取决于其父元素。

navbar-item带有has-dropdown修饰器，有2种可选的修饰器

  • is-hoverable：鼠标指针悬停在父容器navbar-item时，下拉菜单就会显现
  • is-active：下拉菜单会一直显示

当CSS:hover可以完好的工作，is-active修饰器可以由需要使用JavaScript的用户来控制导航条显示与否
